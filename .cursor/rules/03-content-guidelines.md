# 内容编写规范

## 文案逻辑结构

所有页面文案遵循以下叙事链：

```
行业痛点 → 用友产品能力 → 泊冉增值服务 → 业务价值
```

---

## 约束条件

### ✅ 必须
- 效果描述带数据或数据占位符
- 示例：`效率提升 [XX]%`、`实施周期缩短 [N] 天`

### ❌ 禁止
- 无数据支撑的空泛表述（如"提升效率"、"降低成本"）
- 通用商务图库素材

---

## 配图要求

| 类型 | 要求 |
|------|------|
| 产品截图 | 真实系统界面 |
| 架构图 | 标注泊冉增值模块 |
| 客户现场 | 实地拍摄照片 |
| 团队照片 | 真实员工实景 |

---

## 信任背书模块

### 资质展示
- 国家高新技术企业
- 双软认证企业
- 用友铂金伙伴
- 致远互联全国十大定制工厂

### 客户 Logo 墙
- 使用 **灰度** 处理展示
- 按行业分组（制造、零售、服务等）
- "用友铂金伙伴" 文字以红色高亮

### 数据指标
- `12年+` 行业经验
- `500+` 集团型客户
- `100+` 软件著作权
- `98%` 交付成功率
