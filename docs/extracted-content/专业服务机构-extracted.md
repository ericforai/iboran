* 支持商机划分类型，对不同类型可设置不同的业务参数及适用组织、适用角色。

支持基于行动的签到、签退，也支持不基于行动的日常签到、签退管理

做业务现场拜访时，会涉及打卡签到、签退、各种行动业务信息采集，比如采集联系人、竞品价格等信息。

对于业务工作的反馈，企业通过业务员提交日报、周报、月报等提交给业务领导，业务领导根据情况进行相应的指导。

# 商机管理模型

任务：指针对业务工作中自己或别人安排的工作计划。

行动：指为完成业务工作而执行的一系列行为。

跟进记录：指对业务跟进情况的小结，比如对客户拜访、客户跟进情况的小结。

关联对象：

线索、客户、联系人、商机、合同草案

以统一任务、行动模型涵盖线索、客户、终端等的拜访计划及执行的管理需求

# 行为管理—任务

支持定义任务类型，可以对任务类型指定行动类型（行动类型上有设定的行动事项），即支持对不同任务类型制定不同的行动事项要求。

任务类型支持设置各种业务参数

支持对线索、客户、联系人、商机、合同草案等业务进行任务的指定、执行、验收。

支持自己创建任务，也支持指派任务。

业务人员有拜访跟进客户的计划目标，自行制定任务。

领导指派任务给业务员。

对于不同类别的任务有不同的业务要求。

对于一个任务，存在多人执行的情况，并且每个人有自己的任务完成情况，有些任务还会涉及对执行情况的验收。

# 行动及行动事项

支持定义行动类型，类型上可以指定行动事项：比如签到、签退、跟进记录、进销存信息、竞品信息等。

支持针对线索、客户、联系人、商机、合同草案新增行动。（位置：CRM\-行为管理\-行动管理\-行动）

系统既支持基于任务生成行动，也支持不基于任务直接新增行动。

对线索、客户、联系人、商机开展各类行动

有基于任务的行动，也有不基于任务的行动。

对于现场执行任务的业务员，一些企业要求打卡签到、签退，需要现场采集业务数据，比如联系人信息、客户资产信息、客户进销存信息、竞品信息等。

# 跟进记录

支持在线索、客户、联系人、商机、合同草案的详情中填写跟进记录。

支持单独的菜单节点查看查询跟进记录（位置：CRM\-行为管理\-行动管理\-跟进记录）

业务人员对线索、客户、联系人、商机进行拜访跟进，需要填写跟进记录。

# 工作报告

支持工作报告的类型设置：可设提交期间（日报、周报还是月报）、适用的组织角色、提交截止时间、提交提醒。

支持对日报/周报/月报编写草稿及提交

支持查看接收人已阅/未阅状态

支持查看日报/周报/月报时进行评论

支持模板配置日报/周报/月报提交信息

支持根据类型设置里的提交规则控制提交时间、提醒时间

业务员提交工作报告；部门经理收到消息提醒后查看业务员的工作报告；经理对业务员的工作进行评论、指导。

部门经理或内勤人员能够查看工作报告统计情况：谁提交了、谁未提交（注：工作报告统计放在决策分析里）

# CRM数据权限

支持按角色、用户授数据权限

支持主组织权限（数字化建模\-权限管理\-授权\-授权）

CRM业务单据支持按组织、部门、区域、人员、客户维度控制数据权限

人员、部门、区域、组织、客户维度下又支持基于规则的权限范围细化，比如人员权限范围可设置：当前人员、当前人员下属（任职）及下级、当前人员下属（兼职）及下级、当前人员所属销售区域及下级、当前人员所属组织级下级

参与人权限：设置参与人数据权限后，参与人可以查看到其参与的客户、商机等业务数据。



* 不同业务角色要求查看维护不同的数据，典型场景：
  * 集团企管经理/集团商务经理：查看整个集团的业务数据
  * 分公司总经理：查看整个分公司的业务数据
  * 销售总监：查看维护多部门/多区域的业务数据
  * 部门经理/区域经理：查看维护所管辖部门/所管辖区域的业务数据
  * 业务员（客户经理）：查看维护自己做的业务数据
  * 参与人：客户商机等跟进中，一个业务员主推进，还有其他人参与跟进，参与人能看客户商机等数据。
